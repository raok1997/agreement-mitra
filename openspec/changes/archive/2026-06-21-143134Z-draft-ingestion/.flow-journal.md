## Review resolution ‚Äî 2026-06-21  [‚úÖ]
Outcome: User OVERRODE the freeze semantics ‚Äî v1 is **lock-forever** (draft finalized on

is the provider's job on the existing request (CR-4 reconciliation), and a genuine
purchaser revision is a separate PAID supersede flow. Reverted proposal/design D5/spec/
tasks back to `existsByAgreementId` lock-forever. All OTHER review fixes retained.
Decisions: lock-forever for v1; draft-edit-after-signing deferred to a documented future
feature. Created `docs/future-features/` with `index.md` + `draft-revision-after-signing-
request.md` (paid supersede flow; deps: payments module, signing-FSM supersede state,
signing-auth, CR-6 re-stamp). `openspec validate --strict` clean.
