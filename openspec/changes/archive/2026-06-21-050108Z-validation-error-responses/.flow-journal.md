## apply — 2026-06-21  [✅]
Outcome: Implemented the RFC 9457 error contract. New root-package GlobalExceptionHandler
(extends ResponseEntityExceptionHandler), ResourceNotFoundException, FieldErrorDetail record;
AgreementController.get now throws; spring.mvc.problemdetails.enabled=true. Unit + integration
tests added/tightened. Full ./run-tests.sh check green (46 tests incl. 9 integration, 0
skipped/0 failed; jacoco + osvScan + spotbugs/securityScan all pass).
Decisions / discoveries during apply:
- LEAK CAUGHT BY TESTS: Spring auto-populates ProblemDetail.instance with the request path,
  which reflected the requested id and the mistyped path value (not-a-uuid) into the body —
  violating the never-echo invariant. Fixed by pinning instance to a constant (the type URN)
  in the problem() helper; framework only fills a null instance, so ours wins. The POST 400s
  passed only because their path carries no id — the GET tests are what exposed it.
- handleTypeMismatch (the real ResponseEntityExceptionHandler hook) used for
  MethodArgumentTypeMismatchException, per design D6.
- ModularityTests green: signing.api -> root-package ResourceNotFoundException is an allowed
  dependency (root is not a module).
- ENV NOTE: plain `./gradlew check` SKIPS Testcontainers integration tests (no Docker socket
  wired) AND gradle build-cache returns the skipped result as FROM-CACHE; must use
  `./run-tests.sh` (+ --rerun-tasks --no-build-cache when re-verifying) to actually execute them.
- Spotless removeUnusedImports strips a just-added import if its usage isn't in the same edit;
  add import + usage together (bit me 3x on controller + test imports).
